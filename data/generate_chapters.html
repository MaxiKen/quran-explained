📖 Chapter Data — Split Format Guide

Architecture: Two Files Per Chapter

Each chapter is split into two files for maximum loading speed:

File Contains Size When Loaded
data/chapter_XXX.js Theme descriptions, verse numbers, Arabic text, English phrase keys (empty explanation values) Small (~2-30KB) When user clicks the chapter
data/tafsir_XXX.js Explanations/commentary only, keyed by verse number Large (~50-500KB) Prefetched 300ms after chapter renders, OR on-demand when user clicks a phrase

File 1: Chapter Structure — data/chapter_XXX.js

Variable name: var chapterData_X = [...] where X is the chapter number (no padding).

var chapterData_2 = [
  {
    "theme_no": 1,
    "theme_description": "Guidance for the Righteous",
    "verses": [
      {
        "ayah_no_surah": 1,
        "ayah_ar": "الم",
        "ayah_en": {
          "Alif-Lam-Mim": ""       <-- empty string, explanation is in tafsir file
        }
      },
      {
        "ayah_no_surah": 2,
        "ayah_ar": "ذَٰلِكَ الْكِتَابُ لَا رَيْبَ فِيهِ هُدًى لِّلْمُتَّقِينَ",
        "ayah_en": {
          "This is the Book": "",   <-- empty
          "in which there is no doubt": "",
          "a guide for the righteous": ""
        }
      }
    ]
  },
  {
    "theme_no": 2,
    "theme_description": "Characteristics of the Believers",
    "verses": [ ... ]
  }
];

File 2: Tafsir/Commentary — data/tafsir_XXX.js

Variable name: var tafsirData_X = {...} where X is the chapter number.

Structure: Object with verse numbers as keys, each containing phrase-explanation pairs.

var tafsirData_2 = {

  1: {
    "Alif-Lam-Mim": "These are disconnected letters that appear at the beginning..."
  },

  2: {
    "This is the Book": "Allah refers to the Quran as 'the Book' emphasizing...",
    "in which there is no doubt": "The Quran is absolutely free from any doubt...",
    "a guide for the righteous": "Guidance is specifically for those who have taqwa..."
  },

  3: {
    "phrase here": "explanation here...",
    "another phrase": "another explanation..."
  }

};

⚠️ Important Rules

  • Phrase keys MUST match exactly between chapter file and tafsir file (same spelling, same punctuation)
  • Verse numbers must match — the key in tafsir (e.g., 2:) must equal ayah_no_surah in the chapter file
  • Variable naming: chapterData_1 (not chapterData_001), tafsirData_1 (not tafsirData_001)
  • File naming: chapter_001.js, tafsir_001.js (zero-padded to 3 digits)
  • No trailing commas after the last item in objects/arrays (breaks older browsers)

Backward Compatibility

The app supports BOTH formats simultaneously:

  • Split format (chapter + tafsir files): Chapter file has empty explanation strings → app loads tafsir file
  • Old format (single file with inline explanations): Chapter file has non-empty explanation strings → app uses them directly, no tafsir file needed
  • Mixed: Some chapters can be split, others can use old format. The app handles both.

This means you can migrate chapters one at a time without breaking anything.

How Loading Works

  1. User clicks a chapter → chapter_XXX.js is fetched → page renders with Arabic + English phrases
  2. 300ms later → tafsir_XXX.js starts prefetching in background
  3. User clicks a phrase → app checks tafsir cache → if loaded, shows instantly
  4. If tafsir not yet loaded → modal shows "Loading..." → fetches tafsir → updates modal
  5. Service worker caches both files → works offline next time

Creating a New Chapter (Step by Step)

  1. Create data/chapter_XXX.js with themes, verses, Arabic text, and English phrase keys (empty values)
  2. Create data/tafsir_XXX.js with explanations keyed by verse number
  3. Test by opening the chapter in the app — phrases should render, clicking should show explanations
  4. Both files are automatically cached by the service worker for offline use
